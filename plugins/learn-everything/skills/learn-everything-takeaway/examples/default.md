# Example: default takeaway (quality reference)

Constructed example: a blog post arguing that "reading more books" is bad advice.

---

## Takeaway: 为什么"多读书"是一个坏建议

**一句话总结：** 阅读量不是知识积累的瓶颈，内化和输出才是——在没有输出机制的情况下增加输入，只会制造"已读"的幻觉。

### 核心洞见

作者的核心主张：大多数人推荐"多读书"，是把"书读完"等同于"知识获得"。这是一个类别错误（category error）——阅读是信息输入，知识是改变行为或思维的能力，两者之间需要主动加工（写作、讲述、应用）才能转化。

支撑证据：
- 认知科学的"生成效应"（Generation Effect）：主动提取信息的记忆强度是被动阅读的3-4倍
- 作者个人实验：读100本书但不输出，一年后能回忆的不超过5%；读20本书但每本写500字笔记，一年后70%内容仍可提取
- Feynman技巧的核心假设：能用简单语言解释 = 真正理解，否则只是熟悉感

### 什么被系统性忽略了

"多读书"建议假设读者已有内化机制，只是缺乏输入。现实是大多数人缺的是处理机制，不是原材料。增加输入只会放大这个瓶颈。

### 这改变了我什么

以前：用"今年读了多少本"衡量学习进展  
之后：用"今年有多少本书的内容我能在对话中自然引用"来衡量

具体改变：读书时主动记录"这本书最让我改变想法的一句话是什么"，一本书只要有一句这样的句子就值得读完。

### 最值得记住的对比

> "Reading without output is like eating without digesting. The number of meals doesn't matter."

---

## 什么使这个 takeaway 质量达标

- "核心洞见"段落说清楚作者在论证什么逻辑，不只是总结观点
- 支撑证据有具体数字（3-4倍、5% vs 70%），不是"研究表明"
- "系统性忽略"段落指出了文章框架的前提假设
- "这改变了我什么"用前后对比的形式，不是"很有启发"
- 最后的引用有画面感，可以独立流传
