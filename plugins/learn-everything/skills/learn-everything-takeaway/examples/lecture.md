## Takeaway: 动态规划——从递归到记忆化的思维跃迁

**一句话总结：** DP 不是一种算法，而是识别"重叠子问题"的思维框架——看见了，就把指数复杂度压成多项式。
<!-- 为什么这么写: 提炼讲师在论证什么(一个思维框架),不是"讲了 DP"。 -->

### 核心论题

原始递归遇重叠子问题会指数级重复计算；DP 的本质是给递归加"记忆"。Fibonacci：朴素 O(2^n) → 加备忘录 O(n)。
<!-- 为什么这么写: 一段说清"讲师在论证什么",配一个最小演示,不流水账。 -->

### 关键概念（定义 · 用途 · 关系）

- **重叠子问题：** 子问题在递归树里重复出现 → DP 适用的必要条件。
- **最优子结构：** 最优解由子问题最优解构成 → 正确性保证；贪心也要它，但不要求重叠。
<!-- 为什么这么写: 每个概念给"定义+用途+与其他概念的关系",不是孤立罗列;关系(与贪心的异同)才让概念立体。 -->

### 讲师最有价值的一句

> "Every DP problem is just a DAG shortest path problem in disguise."
<!-- 为什么这么写: 直接引用原话、不转述;这种一句话重构整个框架,最该原样保留。 -->

答：画递归树，检查是否有重叠子问题 + 最优子结构。
